# Contributing to Windows 11 Portfolio

Thank you for your interest in contributing! This project is a personal portfolio website built with HTML, CSS, and Vanilla JavaScript, themed after Windows 11.

## How to Contribute

### Reporting Bugs

If you find a bug, please open an issue and include:
- A clear, descriptive title.
- Steps to reproduce the issue.
- Expected and actual behavior.
- Screenshots if applicable.
- Browser and OS information.

### Suggesting Enhancements

Feature suggestions are welcome. Please open an issue with:
- A clear description of the proposed feature.
- Why it would be useful to the project.
- Any relevant examples or references.

### Submitting Pull Requests

1. Fork the repository.
2. Create a new branch from `main`:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. Make your changes, keeping them focused and minimal.
4. Test your changes in a modern browser before submitting.
5. Commit with a clear message describing what you changed.
6. Open a Pull Request against the `main` branch.

### Code Style

- Use 4-space indentation for HTML, CSS, and JavaScript.
- Keep CSS custom properties (variables) consistent with the existing `--win-*` naming convention.
- Avoid introducing external libraries or frameworks — this project intentionally uses Vanilla JS.
- Ensure new UI elements match the Windows 11 Light Mode aesthetic used throughout the project.

## Development

Since this is a static front-end project, no build step is required:

1. Clone the repository.
2. Open `index.html` directly in a browser, or use a local development server such as VS Code's Live Server extension or `npx serve` for full API functionality.
